> Source: Lecture transcript 

---

## 🔹 1. What is Information Retrieval (IR)?

### 📌 Definition

**Information Retrieval (IR)** is the process of:

> Finding relevant information from a large collection of data based on a user's need.

### 🔍 Breaking the Term

* **Information** → Collection of facts used to draw conclusions
* **Retrieval** → Process of searching and finding
* **IR = Information + Retrieval**

👉 Final Meaning:

> Retrieving relevant **unstructured documents** that satisfy a user’s **information need**

---

## 🔹 2. Why IR is Important

### 🌍 Real-world Context

* Massive data growth (websites, books, social media, research papers)
* Traditional search methods are insufficient

### 🚀 Applications

* Search Engines (Google, Bing)
* E-commerce search
* Chatbots & QA systems
* Recommendation systems
* Document retrieval systems

### 🧠 Key Insight

> IR enables efficient **search, discovery, and learning** in a data-rich world

---

## 🔹 3. Key Concepts in IR

### 📄 3.1 Unstructured Data

* No fixed schema (unlike databases)
* Treated as a **stream of text**
* Examples: documents, emails, tweets

---

### 🎯 3.2 Information Need

* The **actual intent** behind a query
* NOT equal to the query itself

| Concept          | Meaning         |
| ---------------- | --------------- |
| Information Need | What user wants |
| Query            | What user types |

---

### 📚 3.3 Corpus

* Collection of documents to search from

---

## 🔹 4. IR System Workflow

### ⚙️ Step-by-Step Process

#### 1. Document Processing

* Tokenization
* Cleaning
* Representation

#### 2. Indexing

* Create efficient data structures (e.g., inverted index)

#### 3. Query Processing

* User submits query
* System interprets it

#### 4. Matching

* Compare query with indexed documents

#### 5. Ranking

* Order documents by relevance

#### 6. Results Display

* Show top relevant documents

#### 7. Feedback Loop

* User interaction improves system

---

## 🔹 11. Evaluation Metrics

| Metric    | Purpose                   |
| --------- | ------------------------- |
| Precision | Correctness of results    |
| Recall    | Coverage of relevant docs |
| F1 Score  | Balance                   |
| MAP       | Mean performance          |
| NDCG      | Ranked relevance          |

## 🔹 12. Key Takeaways

* IR is about **relevance, not exact matching**
* Query ≠ Information Need
* Ranking is the **core problem**
* BM25 is still highly relevant
* Evaluation is essential
* Modern IR uses **ML + Neural Networks**
