# 🔹 1. Formal Definition of IR

### 👨‍🏫 Historical Definition

* Proposed by **Gerard Salton (1960s)**
* Founder of:

  * SMART system
  * Vector Space Model

### 📌 Definition

> IR is the study and practice of how we **structure, analyze, organize, store, and retrieve information**

---

# 🔹 2. Real-World Applications of IR

### 🌍 Everyday Systems Using IR

* 🔎 Web search (Google)
* 🛒 E-commerce search (Amazon)
* 🎬 Recommendation systems (Netflix)
* 💻 Desktop search
* 📚 Digital libraries
* 📱 Social media search
* ❓ Q&A platforms (Stack Overflow)
* 🏢 Enterprise search (company data)
* 📜 Patent search
* 👨‍🔬 Expert search systems

### 💡 Core Idea

> Whenever you search in a large dataset → it’s an IR problem

---

# 🔹 3. IR vs Traditional Library Search

### 📚 Traditional Approach

* Uses:

  * Catalogs
  * Metadata (title, author)
* Works for:

  * Simple queries

### ❌ Limitations

* Cannot handle:

  * Complex queries
  * Content-based search
* Requires:

  * Human expert (librarian)

---

# 🔹 4. IR vs Database Systems (VERY IMPORTANT 🔥)

| Feature     | Information Retrieval | Databases            |
| ----------- | --------------------- | -------------------- |
| Data Type   | Unstructured text     | Structured tables    |
| Query Type  | Natural language      | SQL                  |
| Matching    | Approximate           | Exact                |
| Output      | Ranked results        | All matching results |
| Flexibility | High                  | Low                  |

---

### 📌 Key Insight

> IR = **Approximate + Ranked Search**
> DB = **Exact + Filtered Search**

---

# 🔹 5. Why IR is Difficult

### ⚠️ Core Challenge: Uncertainty

#### 1. User Problem

* Doesn’t know exact query
* Example:

  * "nobel prize india professor" ❌

#### 2. System Problem

* Cannot fully understand language
* Cannot ask clarifying questions

#### 3. Matching Problem

* Query ≠ Document words

#### 4. Satisfaction Problem

* Hard to know if user is satisfied

---

### 💡 Summary

> IR is about making **best possible guesses**

# 🔹 9. Bag of Words Model (BoW)

## 🧠 Core Idea

> Ignore grammar → focus only on words

---

## 🔧 How It Works

### Step 1:

Convert document → list of words

### Step 2:

Convert query → list of words

### Step 3:

Measure overlap

---

## 📊 Example

| Query | "climate change health"             |
| ----- | ----------------------------------- |
| Doc1  | contains all words → high relevance |
| Doc2  | contains 1 word → low relevance     |

---

## 📌 Formula Idea

> More overlap = More relevance

---

## ⚠️ Limitation

* Ignores:

  * Word order
  * Context
  * Meaning

---

# 🔹 11. Term Selection Approaches

---

## 🟢 1. Controlled Vocabulary

### 📌 Definition

* Predefined set of terms
* Assigned by experts

### ✅ Advantages

* Less ambiguity
* Structured

### ❌ Disadvantages

* Expensive
* Limited coverage
* Misses new topics

---

## 🔵 2. Free Text Vocabulary (MOST IMPORTANT)

### 📌 Definition

* Use all words in document

### ✅ Advantages

* Cheap
* Flexible
* Detailed

### ❌ Disadvantages

* Ambiguity
* Noise

---

## 📊 Comparison

| Feature     | Controlled | Free Text |
| ----------- | ---------- | --------- |
| Cost        | High       | Low       |
| Ambiguity   | Low        | High      |
| Detail      | Limited    | High      |
| Flexibility | Low        | High      |

---

### 💡 Industry Reality

> Most modern search engines use **Free Text Vocabulary**

---
