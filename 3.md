# 🔹 1. Term Filtering (Text Processing Pipeline)

After selecting words (free text), we **clean & normalize** them.

---

## ⚙️ Pipeline Steps

### 1️⃣ Tokenization

### 📌 Definition

> Splitting text into smaller units called **tokens (words)**

---

### 🧠 Examples

| Input       | Tokens        |
| ----------- | ------------- |
| "I love IR" | [I, love, IR] |

---

### ⚠️ Challenges

* "USA" → 1 token or 3?
* "3.14" → split or not?
* "Hong Kong" → 1 or 2 tokens?
* Non-English languages (no spaces)

---

### 💡 Key Insight

> Tokenization is **language-dependent**

---

## 2️⃣ Case Folding (Downcasing)

### 📌 Goal

Convert all words → lowercase

---

### 🧠 Example

| Before           | After   |
| ---------------- | ------- |
| Kolkata, KOLKATA | kolkata |

---

### ⚠️ Problems

| Issue        | Example                |
| ------------ | ---------------------- |
| Ambiguity    | Bush (plant vs person) |
| Acronyms     | US → us                |
| Meaning loss | CAT exam → cat animal  |

---

### 💡 Trade-off

> Simplicity > Perfect accuracy (in large systems)

---

## 3️⃣ Stop Word Removal

---

### 📌 Definition

> Remove common words with **low meaning**

---

### 🧠 Examples

* the, is, am, of, a

---

### ✅ Benefits

* Smaller index
* Faster search
* Better relevance

---

### ❌ Risks

| Example              | Problem          |
| -------------------- | ---------------- |
| "to be or not to be" | Meaning lost     |
| "I was not present"  | Meaning reversed |

---

### ⚖️ Strategies

| Approach   | Description       |
| ---------- | ----------------- |
| Aggressive | Remove many words |
| Minimal    | Remove few words  |
| None       | Keep all          |

---

### 💡 Key Insight

> No universal rule — depends on application

---

# 🔹 2. Linguistic Processing (VERY IMPORTANT)

## 🎯 Goal

Handle word variations → improve matching

---

## 🔥 Problem (BoW Limitation)

| Document  | Query       |
| --------- | ----------- |
| institute | institution |

👉 No match (but should match!)

---

## 🧠 Solution

Normalize words to **common base form**

---

## 🔹 Techniques

---

### 🟢 1. Lemmatization

### 📌 Definition

> Convert word → dictionary base form (lemma)

---

### 🧠 Examples

| Word | Lemma |
| ---- | ----- |
| am   | be    |
| cars | car   |

---

### ✅ Pros

* Accurate
* Context-aware

### ❌ Cons

* Slow
* Computationally expensive

---

---

### 🔵 2. Stemming (MORE IMPORTANT)

### 📌 Definition

> Remove suffixes → get root form

---

### 🧠 Examples

| Word      | Stem     |
| --------- | -------- |
| institute | institut |
| running   | run      |

---

### ✅ Pros

* Fast
* Simple

### ❌ Cons

* Not always meaningful words

---

### ⚙️ Types

* Rule-based (Porter Stemmer ⭐)
* Statistical

---

### 💡 Key Insight

> Stemming is widely used in real systems

---

# 🔹 3. Why Indexing is Needed

---

## ❌ Problem with Naive Search

### Linear Scan (grep style)

* Check every document
* Very slow for large data

---

### ⏱ Complexity

> O(N × document size)

---

### ❌ Limitations

* Slow
* Cannot rank
* Cannot handle complex queries

---

### 💡 Conclusion

> Need efficient data structure → **INDEX**

---

# 🔹 4. Term-Document Incidence Matrix

---

## 📌 Definition

Matrix representing:

* Rows → Terms
* Columns → Documents

---

## 📊 Values

| Value | Meaning      |
| ----- | ------------ |
| 1     | Term present |
| 0     | Term absent  |

---

## 📏 Size

> m × n (terms × documents)

---

## ⚠️ Problem

* Too many zeros
* Memory inefficient
* Slow

---

## 📌 Concept

Use logical operators:

---

Query:

```

```

---

## ⚙️ Process

* Convert terms → vectors

---

## ❌ Limitation

* Only exact matches

# 🔥 6. Inverted Index (MOST IMPORTANT TOPIC)

---

## 📌 Definition

> Maps **term → list of documents**

---

## 🧠 Idea

Instead of:

❌ term → doc matrix

Use:

✅ term → doc list

---

## 📊 Structure

### 1. Dictionary

* List of all terms (sorted)

### 2. Posting Lists

* For each term → list of doc IDs

---

### 🧠 Example

```
batman → [2, 8]
alfred → [1, 4, 7]
```

---

## 🔁 Why "Inverted"?

| Type           | Mapping     |
| -------------- | ----------- |
| Forward Index  | Doc → Terms |
| Inverted Index | Term → Docs |

---

# 🔹 7. Query Processing (Inverted Index)

---

## 🔍 Example Query

```
alfred AND bruce
```

---

## ⚙️ Steps

1. Get postings:

   * alfred → [1, 2, 8]
   * bruce → [2, 8, 37]

2. Merge lists

---

## 🧠 Merge Algorithm

* Use two pointers
* Traverse both lists

---

## ⏱ Complexity

> O(x + y)

---

# 🔹 8. Optimization Techniques

---

## 📌 1. Document Frequency (df)

> Number of documents containing term

---

### 💡 Usage

* Process rare terms first
* Faster intersection

---

## 📌 2. Query Optimization Rule

> Process terms in increasing **df**

---

### 🧠 Example

```
alfred (df=8)
batman (df=4)
bruce (df=7)
```

👉 Order:

```
batman → bruce → alfred
```

---

## 📌 3. Skip Pointers

---

### 📌 Idea

Jump ahead in posting lists

---

### ✅ Benefit

* Faster merging

---

### ⚠️ Trade-off

* More skips = more memory
